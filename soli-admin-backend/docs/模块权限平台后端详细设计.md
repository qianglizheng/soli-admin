# 模块权限平台后端详细设计

## 1. 文档定位

本文档面向以下 5 个后台管理模块的后端落地设计：

1. 模块中心
2. 岗位管理
3. 功能授权
4. 字段标题
5. 状态权限

设计目标：

- 与当前 `soli-admin-backend` 工程结构保持一致。
- 支持企业级组织岗位、多模块、多状态、字段级和按钮级权限。
- 后台管理端严格按需加载，避免一次性加载全部字段标题、全部岗位人员、全部模块字段权限。
- 运行时仍然允许按“单模块、单岗位、单状态”一次装配上下文，保证前端渲染简单。

本文档是对 [单据模块权限平台重构设计.md](/E:/devProjects/soli-admin/soli-admin-backend/docs/单据模块权限平台重构设计.md) 的后端细化版，重点补充表结构、JavaBean、接口、缓存和按需加载策略。

## 3. 核心设计原则

### 3.1 授权主体是组织岗位，不是角色

- 角色继续负责登录入口、菜单入口、通用接口权限。
- 单据模块内部权限的授权主体统一为 `组织岗位`。
- 同名岗位在不同组织下是不同授权主体。

### 3.2 模块元数据、标题、岗位权限、状态权限解耦

- 模块中心维护技术元数据。
- 字段标题维护展示文案，不直接修改权限。
- 功能授权维护岗位基线权限。
- 状态权限维护状态维度附加限制。
- 运行时上下文负责把上述信息装配成前端可消费结构。

### 3.3 状态权限只允许收紧

- 字段权限级别：`0 = hidden`，`1 = read`，`2 = write`
- 按钮权限级别：`0 = hidden`，`1 = disabled`，`2 = enabled`
- 最终权限统一按 `min(岗位基线, 状态限制)` 计算。

### 3.4 后台管理端必须按需加载

后台管理端严禁设计以下接口：

- 一次性返回某岗位全部员工明细
- 一次性返回全部模块的全部字段标题
- 一次性返回某岗位对全部模块的全部字段权限
- 一次性返回某模块全部状态、全部 Tab、全部字段、全部按钮的完整权限明细

后台管理端的加载规则：

- 树只加载树本身和摘要字段。
- 选中节点后再加载详情。
- 字段列表按 `模块 + Tab + 分页` 查询。
- 岗位人员按 `岗位 + 分页` 查询。
- 权限编辑按 `岗位 + 模块 + Tab/Area + 分页` 查询。

### 3.5 运行时上下文允许单模块一次装配

这不是“全量加载”，因为范围仅限：

- 单个模块
- 当前岗位
- 当前状态
- 当前上下文版本

运行时一次装配是合理的，因为业务页面进入单个模块时，前端需要完整上下文完成渲染与写操作校验。

## 4. 按需加载总策略

| 模块 | 首屏只加载 | 选中后再加载 | 禁止一次性加载 |
| --- | --- | --- | --- |
| 岗位管理 | 组织岗位树、岗位摘要计数 | 岗位详情、岗位员工分页、可选员工分页 | 某岗位全部员工、全部可选员工 |
| 模块中心 | 模块树、模块摘要计数 | 模块基础信息、Tab 摘要、字段分页、按钮列表、状态列表 | 全模块树下所有字段 |
| 功能授权 | 岗位树、模块树 | 指定岗位+模块的模块权限摘要、按钮列表、Tab 摘要、字段权限分页 | 某岗位全部模块全部字段权限 |
| 字段标题 | 模块树 | 指定模块的 Tab 摘要、字段标题分页 | 全部模块全部字段标题 |
| 状态权限 | 状态型模块树 | 指定模块的状态列表、流转列表、按钮限制列表、字段限制分页 | 某模块全部状态全部字段限制 |

推荐分页默认值：

- `pageNum = 1`
- `pageSize = 20`

建议限制：

- 字段分页最大 `pageSize = 100`
- 岗位员工分页最大 `pageSize = 100`
- 可选员工弹窗最大 `pageSize = 50`

## 5. 表结构详细设计

## 5.1 公共约定

### 5.1.1 主键策略

当前 `BaseCrudServiceImpl` 使用 `YitIdHelper.nextId()` 生成主键，因此新增业务表建议统一使用：

- `id bigint(20) not null`

不建议依赖数据库自增作为主键来源。

### 5.1.2 审计字段

除纯关联表外，建议统一包含以下审计字段：

- `create_by varchar(32)`
- `create_time datetime`
- `update_by varchar(32)`
- `update_time datetime`
- `note varchar(500)`

### 5.1.3 状态字段

主数据表统一使用：

- `status char(1)`，`0 = 启用`，`1 = 停用`

### 5.1.4 外键策略

仓库当前为 MyBatis + 逻辑外键风格，建议：

- 数据库层不强制建立外键约束
- 通过唯一索引、普通索引和 Service 校验保证数据一致性

## 9. 事务、缓存、版本、审计

## 9.1 事务边界

### 9.1.1 岗位管理

- 新增岗位：单事务
- 绑定岗位员工：单事务批量写 `sys_user_org_post`
- 删除岗位：必须先校验无子岗位、无员工绑定、无权限配置

### 9.1.2 模块中心

- 保存模块基础信息：单事务
- 保存 Tab：单事务
- 保存字段：单事务
- 保存按钮：单事务
- 保存状态和流转：单事务
- 任一保存成功后，递增模块 `context_version`

### 9.1.3 功能授权 / 字段标题 / 状态权限

- 批量保存当前页数据时使用单事务
- 保存后递增模块 `context_version`
- 写 `sys_module_publish_log`

## 9.2 缓存设计

### 9.2.1 后台管理端

后台管理端不建议重缓存明细数据，只建议缓存：

- 岗位树
- 模块树
- 状态型模块树

缓存失效条件：

- 树节点新增、修改、删除后清理对应树缓存

### 9.2.2 运行时上下文

建议缓存 Key：

- `module_ctx:{moduleCode}:{orgPostId}:{stateCode}:{contextVersion}`

缓存值：

- `SysModuleContextDTO` 的 JSON 快照

失效策略：

- 只要 `context_version` 变化，天然失效旧 Key

## 9.3 版本策略

`sys_module.context_version` 需要在以下场景递增：

- 模块基础信息修改
- Tab 修改
- 字段修改
- 按钮修改
- 状态修改
- 流转修改
- 字段标题修改
- 岗位字段权限修改
- 岗位按钮权限修改
- 岗位模块权限修改
- 状态字段权限修改
- 状态按钮权限修改

## 9.4 审计建议

建议记录以下操作日志：

- 岗位创建、修改、删除
- 岗位员工绑定、解绑
- 模块、Tab、字段、按钮、状态、流转增删改
- 功能授权批量保存
- 字段标题批量保存
- 状态权限批量保存

## 10. 关键后端校验

### 10.1 岗位管理

- 同组织下岗位编码唯一
- 不允许岗位挂到自身或子孙岗位下
- 删除岗位前必须校验无子节点、无人员、无授权

### 10.2 模块中心

- 模块编码全局唯一
- 同模块下 Tab 编码按 `tab_scope` 唯一
- 同模块下字段编码唯一
- 同模块下按钮编码唯一
- 状态型模块必须存在 `state_field_code`

### 10.3 功能授权

- 非状态型模块也可配置岗位权限
- 不存在字段/按钮的权限项禁止保存
- 模块不可见时，`nav_visible` 必须自动置为不可见

### 10.4 状态权限

- 仅状态型模块允许配置状态权限
- 状态权限级别不得放大岗位权限
- 删除状态前必须校验无流转引用

## 11. 建议落地顺序

1. 先落表和 Mapper XML
2. 再落岗位管理和模块中心基础 CRUD
3. 然后落功能授权分页查询和批量保存
4. 再落字段标题分页查询和批量保存
5. 最后落状态权限和运行时上下文装配

## 12. 本设计对“不要一次性加载”的明确约束

本次设计明确要求：

- 岗位管理页的员工列表必须分页，不允许在岗位树接口中内嵌员工数组。
- 字段标题页必须按 `模块 + Tab + 分页` 加载字段标题，不允许返回全部字段标题。
- 功能授权页必须按 `岗位 + 模块 + Tab/Area + 分页` 加载字段或按钮权限，不允许一次性返回全部模块全部字段权限。
- 状态权限页必须按 `模块 + 状态 + Tab/Area + 分页` 加载限制项，不允许一次性返回全部状态全部字段限制。
- 只有运行时 `module-context` 接口允许针对“当前模块”一次性组装完整上下文。
